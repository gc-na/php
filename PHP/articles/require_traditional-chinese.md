# PHP 中的 require 語句：用於引入檔案的關鍵指令

## 簡介
在 PHP 中，`require` 是一個用於引入和執行指定檔案的語句。當需要將外部代碼或配置檔案整合到當前 PHP 腳本中時，`require` 是一個常用且重要的工具。

## 文檔說明
### 目的
`require` 指令的主要目的是將外部檔案的內容加載到當前檔案中。這使得代碼可重用性提高，並且有助於將代碼組織得更好。

### 用法
`require` 的語法如下：
```php
require '檔案路徑';
```
當指定的檔案無法找到或無法讀取時，`require` 會產生一個致命錯誤，並停止執行腳本。

### 詳細說明
- **路徑**：`require` 可以使用相對路徑或絕對路徑來指定檔案的路徑。
- **檔案類型**：被引入的檔案可以是任何 PHP 檔案，甚至是 HTML 檔案。
- **多次引入**：如果同一檔案被多次引入，`require` 會出現致命錯誤。若希望避免此情況，可以使用 `require_once`，它會檢查檔案是否已經被引入過。

## 示例
以下是 `require` 的基本使用範例：

### 示例 1：基本用法
```php
// main.php
require 'config.php';
echo "配置檔案已成功引入。";
```

### 示例 2：使用 require_once 避免重複引入
```php
// main.php
require_once 'config.php';
require_once 'config.php'; // 不會再次引入
```

## 解釋
### 常見的陷阱
1. **檔案路徑錯誤**：若指定的檔案路徑錯誤，`require` 將導致致命錯誤，建議使用絕對路徑來避免此問題。
2. **代碼可讀性**：過多的 `require` 語句會使代碼變得雜亂，因此應該保持適當的檔案組織結構。
3. **環境依賴**：在不同的伺服器環境中，檔案路徑可能會有所不同，這可能會影響 `require` 的正常運作。

## 單句總結
`require` 是 PHP 中用於引入外部檔案的語句，可提高代碼的可重用性與組織性。
